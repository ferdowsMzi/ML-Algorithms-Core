# ID3 Decision Tree vs. PRISM Rule Learner

This repository features a complete, from-scratch implementation of two classic symbolic machine learning algorithms: the **ID3 Decision Tree** and the **PRISM Rule-Learning** algorithm. This project explores the mechanics of tree-based and rule-based classification. It includes a rigorous comparison of different node-splitting criteria and an automated grid search for hyperparameter tuning.

## Mathematical Foundations: Splitting Criteria

The core of the ID3 algorithm relies on evaluating the "purity" of data subsets. This project implements three distinct mathematical metrics to determine the optimal feature for splitting the data:

### 1. Entropy & Information Gain

- **Entropy:** Measures the amount of uncertainty or impurity in a set of examples.  
  `H(S) = -sum(p_i * log2(p_i + 1e-9)) for i = 1 to c`  
  *Where `p_i` is the proportion of examples belonging to class `i` in set `S`, and a small epsilon `1e-9` is added to prevent `log(0)`.*

- **Information Gain:** Calculates the reduction in entropy after splitting the dataset `S` on a specific feature `A`.  
  `Gain(S, A) = H(S) - sum((|S_v| / |S|) * H(S_v)) for v in Values(A)`

### 2. Gain Ratio

- **Split Information:** Measures the intrinsic information generated by splitting on feature `A`.  
  `SplitInfo(S, A) = -sum((|S_v| / |S|) * log2(|S_v| / |S|)) for v in Values(A)`

- **Gain Ratio:** Normalizes Information Gain to reduce bias toward features with many distinct values.  
  `GainRatio(S, A) = Gain(S, A) / (SplitInfo(S, A) + 1e-9)`

### 3. Gini Impurity

- **Gini Impurity:** Measures the probability of incorrectly classifying a randomly chosen element if it were labeled according to the class distribution in the subset.  
  `Gini(S) = 1 - sum(p_i^2) for i = 1 to c`

The split metric based on Gini chooses the feature that maximizes the reduction in impurity, similar to Information Gain.

## Algorithms Implemented

### ID3 (Iterative Dichotomiser 3)

A top-down, greedy decision tree algorithm.

- **Recursive Splitting:** At each node, the algorithm evaluates all available features using one of the selected metrics (`entropy`, `gain_ratio`, or `gini`) and chooses the one that maximizes purity.
- **Tree Pruning / Stopping Criteria:** Early stopping is implemented using `max_depth` and `min_samples` to reduce overfitting.
- **Robust Prediction:** The `predict_tree` function handles unseen branches by defaulting to the majority class of the parent node.

### PRISM (Sequential Covering)

Unlike decision trees, PRISM learns a modular set of classification rules.

- **Rule Generation:** The algorithm iterates through each target class and greedily constructs rules of the form `IF feature = value AND ...` that maximize purity for that class.
- **Coverage:** Once a rule is formed, the instances covered by that rule are removed, and the process repeats until all instances of the target class are covered.

## Experimental Setup & Evaluation

The project evaluates both models using the **F1-score** to account for potential class imbalance.

1. **Grid Search Tuning:** A custom grid search evaluates the ID3 model across different parameter combinations:
   - `criteria`: `['entropy', 'gain_ratio', 'gini']`
   - `max_depth`: `[3, 5, 9, 11]`
   - `min_samples`: `[2, 10, 20, 30]`

2. **Performance Comparison:** The models are evaluated on a separate validation set, with results saved automatically to `grid_search_results.csv`. Final test evaluations compare the best parameters for each criterion, such as identifying that `gain_ratio` at depth `9` achieved the highest Test F1-score of approximately `0.9398`.

3. **Rule Extraction:** The PRISM rules are ranked by coverage and saved directly to `prism_rules_final.csv` for interpretability.
